Html Elements

Basic HTML Structure

  • <!DOCTYPE html> → defines the document type (HTML5)
  • <html> → root element of an HTML page
  • <head> → contains metadata, title, links to stylesheets or scripts
  • <body> → contains all visible page content
  • <main> → represents the main content of the page (only one per page)
  • <a> → anchor element, creates a hyperlink using the href attribute
  • <p> → defines a paragraph
  • <h1> to <h6> → define headings, from largest to smallest
  • <img> → embeds an image into the page

HTML elements are the fundamental building blocks of the web. They provide the structure and meaning of a document by combining tags, attributes, and content. An element usually consists of an opening tag, optional attributes, the content itself, and a closing tag. For example, the <p> element defines a paragraph, while the <a> element creates a hyperlink that connects one resource to another. Each element has a semantic purpose, which improves both accessibility and search engine optimization (SEO). By correctly using elements, developers ensure that browsers, assistive technologies, and search engines can all interpret the document in a consistent and meaningful way.

Head and Meta

The head element contains information about the page that is not displayed directly. Meta tags provide details such as character encoding, viewport settings, and description for search engines.

  • <title> → sets the page title shown in the browser tab
  • <meta charset="UTF-8"> → defines character encoding
  • <meta name="viewport" content="width=device-width, initial-scale=1.0"> → makes the page responsive
  • <link rel="stylesheet" href="styles.css"> → connects an external CSS file
Forms

Forms are used to collect user input. Labels can be connected to inputs by using the for attribute, which improves accessibility. Radio buttons are grouped by giving them the same name attribute, so the user can only select one option. Checkboxes allow multiple choices. An option can be pre-selected with the checked attribute, and required makes the input mandatory before submitting.

  • <form>→ container for input elements
  • <input type="text">→ single-line text field
  • <input type="email">→ field that expects an email address
  • <input type="radio">→ radio button, select one option
  • <input type="checkbox">→ checkbox, select multiple options
  • <label>→ label for an input field
  • <textarea>→ multi-line text field
  • <select>→ dropdown menu
  • <button type="submit">submits the data form
Semantic Tags

Semantic HTML refers to elements that clearly describe their meaning and role in the structure of a webpage. Instead of using generic tags like <div> or <span>, semantic tags such as <article>, <nav>, and <footer> improve readability, accessibility, and SEO.

Why Semantic HTML?

Semantic HTML elements clearly describe their meaning in a human- and machine-readable way. They improve accessibility, help with SEO, and make your code easier to read and maintain. Examples include: <header>, <nav>, <article>, and <footer>.

Css Elements

What is CSS?

CSS (Cascading Style Sheets) is used to style and visually arrange HTML elements on a web page. It controls layout, colors, fonts, spacing, and much more.

  • Common CSS Syntax
            selector {
             property: value;
            }
Color and Font

Color in CSS

  • color → sets the text color
  • background-color → sets the background color of an element

    • Colors can be defined using:

  • Named colors → red, blue, green, etc.
  • Hex values → #FF5733
  • RGB → rgb(255, 87, 51)
  • HSL → hsl(14, 100%, 60%)
  • Transparent → rgba, hsla (with alpha channel for opacity)

Font Styling in CSS

  • font-family → defines which font to use
  • font-size → sets the size of text (use px, em, rem)
  • font-weight → controls thickness (normal, bold, 100–900)
  • font-style → makes text italic, normal, or oblique
  • line-height → controls spacing between lines
  • text-transform → changes text case (uppercase, lowercase, capitalize)
  • letter-spacing → space between letters
  • text-align → aligns text inside its box

Typography and Units

text-align – Common Values

Used to align text inside a block-level element.

left → aligns text to the left (default in LTR languages)

right → aligns text to the right

center → centers the text

justify → stretches lines so that each line has equal width (like a newspaper)

px, em , rem Short Summary

  px → absolute unit

Fixed size, does not scale with screen or parent element

Example: font-size: 16px;

em → relative to the parent element’s font-size

Example: if parent is 16px, then 1.5em = 24px

Can lead to compounding (nested elements)

rem → relative to the root element’s (html) font-size

More predictable than em

Example: if html { font-size: 16px; }, then 1.5rem = 24px everywhere

  html {
  font-size: 16px;
}

h1 {
  font-size: 2rem; /* = 32px */
}

p {
  font-size: 1em; /* = 16px if inherited from body */
}
Box Model

The CSS box model is a fundamental concept in web design. It defines how the size and spacing of every HTML element is calculated. Each element is treated as a rectangular box made up of four areas: content, padding, border, and margin. Understanding the box model is essential for accurate layout and spacing, as even small changes in padding or margin can affect how elements align on the page.

Positioning

CSS positioning allows you to control exactly where an element appears on the page. By using the position property, you can choose how an element is placed in relation to its normal position or other elements. There are five main position values: static (default), relative, absolute, fixed, and sticky. Each one behaves differently depending on the context and parent containers.

static(default)

This is the default position. The element is placed in the normal document flow and is not affected by top, right, bottom, or left properties.

relative

The element stays in the normal document flow, but you can shift it using top, right, bottom, or left — relative to its original position.

absolute

The element is removed from the normal flow and positioned relative to the nearest positioned (non-static) ancestor. If none exists, it's placed relative to the element.

fixed

The element is positioned relative to the browser window. It stays in the same place even when the page is scrolled.

sticky

The element toggles between relative and fixed depending on the scroll position. It sticks to a defined position when you scroll past it.

Layout

CSS layout techniques allow developers to organize and position elements on a webpage in a meaningful and visually appealing way. Layout determines how elements are sized, spaced, and aligned in relation to one another and to the page itself. A strong layout is essential for usability, readability, and responsive design.

In the early days of the web, layout was mostly handled using floats and positioning, but modern CSS provides powerful tools like Flexbox and Grid, which make complex layouts easier to build and maintain. These modern layout systems are designed to handle different screen sizes and orientations without relying on JavaScript hacks or excessive media queries.

Flexbox is best suited for one-dimensional layouts — either a row or a column — and it’s great for aligning elements dynamically even when their sizes change. CSS Grid, on the other hand, is designed for two-dimensional layouts, allowing you to define rows and columns and place items precisely within them.

Another important concept in layout is the box model, which defines how margins, borders, padding, and content interact. Understanding how these properties affect spacing is essential for creating clean, professional interfaces.

Responsive Design

Responsive design is a web development approach that ensures websites look and function well on all screen sizes — from mobile phones to large desktop monitors. It adapts the layout and content based on the device's screen width, making the site accessible and user-friendly for everyone.

A responsive website typically uses relative units (like %, em, rem), media queries, and flexible grid systems to adjust elements dynamically. Instead of creating separate versions for each device, responsive design allows a single codebase to serve all users.

One key technique is using the <meta name="viewport"> tag in the HTML <head>, which helps control how the page is displayed on mobile browsers. Combined with CSS media queries, this allows developers to change font sizes, rearrange layouts, or hide/show elements based on screen size.

  • <meta name="viewport" content="width=device-width , initial-scale=1.0">
Pseudo Selectors

Pseudo-selectors are special keywords added to selectors that let you target specific states or parts of elements — without having to add extra classes or IDs.
They help you style things like:

  • when a user hovers over a button
  • or select only the first paragraph in a container
  • or insert text before/after an element, just with CSS

Pseudo classes that starts with :

These describe a special state of an element (like hover, focus, visited...).

:hover

Applies when the user’s mouse is over an element.

          button:hover {
            background-color: lightblue;
          }

:active

Applies when the element is being clicked (active state).

         button:active {
           transform: scale(0.95);
         }

:focus

Used for inputs or buttons that have been focused (e.g., clicked or tabbed to).

         input:focus {
           border: 2px solid dodgerblue;
         }

:first-child / :last-child

Targets the first or last child element inside a parent.

  p:first-child {
   color: red;
  }

Pseudo-Elements That Starts with ::

These let you style specific parts of an element (not the whole)

::before , ::after

Let you insert content before or after an element — often used for decoration.

  h1::before {
   content: "★ ";
  }

::first-letter

Targets the first letter of a block of text.

  p::first-letter {
  font-size: 2rem;
  color: crimson;
}

::selection

Applies styles to selected (highlighted) text.

  ::selection {
    background: yellow;
    color: black;
  }
